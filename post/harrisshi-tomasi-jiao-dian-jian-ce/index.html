Harris、Shi-Tomasi角点检测

2020-07-12 16 min read

       由于物体上的点是无穷多的，为了准确描述物体在两幅图像中的位置变化，我们通常只讨论那些特别显著的点，该点能够比较容易且准确地在另一幅图像中找到相匹配，也就是代表了物理空间中同一个位置的点，我们称之为特征点（feature points）。图像中的结构简单地可分为三种，即平坦区域（flat）、边缘（edge）和角（corner），如图 1 所示。很明显，我们能够非常容易地找到角的顶点在两幅像中的对应位置，即便图像发生了缩放、平移、旋转等等变换。但对于那些平坦区域以及边缘上的点，我们却很难去进行匹配，因为所有的点都是相似的。因此，特征点的检测可以转换为对图像角点（corner points）的检测，目前也有很多现成的检测算法，例如 FAST、SUSAN、Harris 等等，其判断角点的依据以及检测的速度、准确率等各有不同，这里我们主要讨论 Harris 角点，以及其一个比较简单的变体 Shi-Tomasi 角点。

图1 角点检测示意图


       Harris 角点主要基于一个较小窗口移动时所产生的灰度变化来描述。如图 1 所示，当窗口处于平坦区域时，无论其往哪个方向移动，灰度都基本不会产生变化；当其处于边缘区域时，沿着平行于边缘的方向移动，灰度同样不会产生太大变化，而当其沿着垂直于边缘方向移动时，灰度则会产生较大的变化；当窗口处于角区域时，无论其往哪个方向移动，灰度都会产生比较大的变化。因此，利用这种特点，我们就可以判断哪些点是我们所需要的角点。通常我们假设图像的灰度值是一个关于像素坐标 (x,y)(x, y)(x,y) 的函数 I(x,y)I(x, y)I(x,y)，那么定义窗口产生了微小位移 (u,v)(u, v)(u,v) 时所造成的灰度变化为

尽管 Sobel 算子看上去只是简单的 [1,2,1][1, 2, 1][1,2,1] 的加权，但其作者 Irwin Sobel 有提到该算子实际上是考虑了梯度的各向同性而推导出来的，也有资料提到其实际上是中心差分法和高斯滤波的结合，而对于更大的如 5×55\times55×5 的 Sobel 算子，网上反而能够找到几个不同的结果，甚至也找不到一个定论。对于 3×33\times33×3 的 Sobel 算子，OpenCV 文档中有提到其效果不是很好，所以一般使用 Scharr 算子，其稍微提高了所在行或列的权重，并且能够获得更高的准确度。

       回到角点检测的问题，对于式（2），我们该怎么判断一个点是不是角点？很明显这并不是一个优化问题，我们要做的是从式（2）来判断当窗口往任意方向移动时，EEE 的变化是否强烈，这样我们才能知道该点是否可能为角点。从式（3）可以知道，M\bf{M}M 是一个对称半正定矩阵，那么其特征值一定为非负，同时其特征向量组成的矩阵是一个正交矩阵，并且有

(5)M=VΛV−1=VΛVT,whereΛ=diag(λ1,λ2),V=(v1v2).{\bf{M}} = {\bf{V}}\Lambda {{\bf{V}}^{ - 1}} = {\bf{V}}\Lambda {{\bf{V}}^T},{\rm{where }}\Lambda = diag\left( {{\lambda _1},{\lambda _2}} \right),{\rm{ }}{\bf{V}} = \left( {\begin{array}{c} {{{\bf{v}}_1}}&{{{\bf{v}}_2}} \end{array}} \right). \tag{5}M=VΛV−1=VΛVT,whereΛ=diag(λ1​,λ2​),V=(v1​​v2​​).(5)

(6)E(u)=uTMu=uTVΛVTu=(VTu)TΛ(VTu)=λ1∥v1Tu∥2+λ2∥v2Tu∥2≤(λ1+λ2)∥u∥2.\begin{array}{c} E\left( {\bf{u}} \right) = {{\bf{u}}^T}{\bf{Mu}} = {{\bf{u}}^T}{\bf{V}}\Lambda {{\bf{V}}^T}{\bf{u}} = {\left( {{{\bf{V}}^T}{\bf{u}}} \right)^T}\Lambda \left( {{{\bf{V}}^T}{\bf{u}}} \right)\\ = {\lambda _1}{\left\| {{\bf{v}}_1^T{\bf{u}}} \right\|^2} + {\lambda _2}{\left\| {{\bf{v}}_2^T{\bf{u}}} \right\|^2} \le \left( {{\lambda _1} + {\lambda _2}} \right){\left\| {\bf{u}} \right\|^2}. \end{array} \tag{6}E(u)=uTMu=uTVΛVTu=(VTu)TΛ(VTu)=λ1​∥∥​v1T​u∥∥​2+λ2​∥∥​v2T​u∥∥​2≤(λ1​+λ2​)∥u∥2.​(6)

因此，我们可以分别进行分析：

  1. 当 λ1，λ2λ_1，λ_2λ1​，λ2​ 都比较小甚至为零时，无论窗口往哪个方向移动，E(u)E(\bf{u})E(u) 都会保持比较小的量级，也就是窗口内的亮度基本没有变化，因此这时候可以判断该点很有可能处在一个比较平坦的区域。
  2. 当 λ1>>λ2λ_1>>λ_2λ1​>>λ2​ 时，根据式（6），由于 v1，v2{\bf{v}}_1，{\bf{v}}_2v1​，v2​ 相互正交，因此当选择 u\bf{u}u 平行于 v1{\bf{v}}_1v1​ 时，其第二项为零，而第一项达到最大，且由于 λ1λ_1λ1​ 比较大，因此 E(u)E(\bf{u})E(u) 也会比较大；相反，因此当选择 u\bf{u}u 平行于 v2{\bf{v}}_2v2​ 时，第一项为零，第二项虽然达到最大，但由于 λ2λ_2λ2​ 比较小，因此 E(u)E(\bf{u})E(u) 依然比较小，或者说相比于前者量级要小很多，具有明显的方向性。从这里我们可以看出，当窗口沿着平行于 v2{\bf{v}}_2v2​ 的方向移动时，其灰度基本没有变化，但沿着其他方向移动时，灰度的变化会比较强烈，特别是沿着垂直于 v2{\bf{v}}_2v2​ 的方向时。因此，我们可以判断该点可能处在一个沿着 v2{\bf{v}}_2v2​ 方向的边缘附近。反之，当 λ2>>λ1λ_2>>λ_1λ2​>>λ1​ 时亦然。
  3. 当 λ1，λ2λ_1，λ_2λ1​，λ2​ 都比较大，并且量级相当时，无论窗口往哪个方向移动，其灰度都会产生强烈的变化，并且没有明显的方向性。因此这时我们可以判断该点很有可能处在一个角点附近。

从另外一个角度分析，当设定 E(u)E(\bf{u})E(u) 为常量时，式（2）实际代表了一个椭圆，其两个半轴的长度分别是矩阵 M\bf{M}M 的两个特征值的倒数的平方根。因此以上（1）的情形代表了一个很大的偏向于正圆的椭圆，u\bf{u}u 的选择有很大的自由度。而（2）的情形代表了一个十分扁平的椭圆，因此 u\bf{u}u 的选择具有很明显的方向性。而（3）的情形则代表了一个很小的偏向于正圆的椭圆，也就是 ∣∣u∣∣||\bf{u}||∣∣u∣∣ 必须要很小才能满足 E(u)E(\bf{u})E(u) 的设定，这与上面的分析是一致的，如图 2 所示。

       综上，我们可以通过分析 M\bf{M}M 的两个特征值来判断一个点是否为角点。例如，我们可以为每个需要检测的点定义一个响应值 RRR，其中 RRR 与 M\bf{M}M 的特征值相关。一般来说 RRR 越大，该点就越可能是角点，这样我们就可以选取那些具有最大的 RRR 值的检测点作为最终的角点。很明显，最简单的方法是令

(7)R=min⁡{λ1,λ2}.R = \min \left\{ {{\lambda _1},{\rm{ }}{\lambda _2}} \right\}. \tag{7} R=min{λ1​,λ2​}.(7)

这就是 Shi-Tomasi 角点检测方法，其实际上是 Harris 角点检测方法的稍微改进，但一般来说能够获得更好的效果。然而，式（7）看上去虽然简单，但实际上求解特征值需要更多的计算步骤，而且其无法从 RRR 值来区分平坦与边缘区域，因此 Harris 最初反而没有直接使用特征值，而是令

(8)R=det⁡(M)−k(Tr(M))2=λ1λ2−k(λ1+λ2)2,whereλ=12(Tr(M)±Tr(M)−4det⁡(M)).\begin{array}{l} R = \det \left( {\bf{M}} \right) - k{\left( {Tr\left( {\bf{M}} \right)} \right)^2} = {\lambda _1}{\lambda _2} - k{\left( {{\lambda _1} + {\lambda _2}} \right)^2},\\ {\rm{where }}\lambda = \frac{1}{2}\left( {Tr\left( {\bf{M}} \right) \pm \sqrt {Tr\left( {\bf{M}} \right) - 4\det \left( {\bf{M}} \right)} } \right). \end{array} \tag{8}R=det(M)−k(Tr(M))2=λ1​λ2​−k(λ1​+λ2​)2,whereλ=21​(Tr(M)±Tr(M)−4det(M)​).​(8)

其中 kkk 是一个自由参数，用于控制角点检测的灵敏度，kkk 越大，响应值 RRR 就会越小，从而减少检测到的角点的个数，一般设为 0.04～0.06。可以看到，Harris 方法并不需要直接求出特征值，省去了开方的运算操作。同时，通过 RRR 值，我们同样可以区分一个像素点属于平坦、边缘和角三种结构中的哪一种，如图 3 所示：

  1. ∣R∣|R|∣R∣ 比较小，这种情况出现在 λ1，λ2λ_1，λ_2λ1​，λ2​ 都比较小的时候，因此这时可以判断该点处于平坦区域。
  2. R<0R<0R<0 且 ∣R∣|R|∣R∣ 比较大，这种情况出现在 λ1>>λ2λ_1>>λ_2λ1​>>λ2​ 或者相反的时候，因此这时可以判断该点处于边缘区域。
  3. R>0R>0R>0 且 RRR 比较大，这种情况出现在两个特征值都比较大且量级差不多的时候，因此这时可以判断该点为角点。
图3 Harris角点响应值分布


       由于这种检测方法是基于一个窗口内像素灰度值的变化来确定的，如果某个检测点被判定为角点，那么其附近的像素点同样很有可能被判定为角点，因为那些窗口同样很有可能包含了那个角。因此，用于角点检测的窗口不适宜设置得太大。另外，还应注意到，这种检测方法并不具有尺度不变性，如图 4 所示，一个曲率比较小的圆角，在使用较小的窗口进行检测时只能检测到边缘，而经过缩小后使用同样的窗口，却能检测出角点，因此对于那些分辨率比较高的图像，我们可以适当增大窗口的尺寸。除此以外，我们也可以通过非极大值抑制的方法来排除那些聚集在一起的角点，因为通常来说真正的角点的响应值应该是最大的。目前已经有不少开源工具库提供了这么一种角点检测方法，比如 OpenCV 中的 goodFeaturesToTrack() 函数，不过该函数只能达到像素级精度，如果需要更高精度的可以使用 cornerSubPix()。
